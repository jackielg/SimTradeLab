## 一、项目规则配置

### 1.1 编码规范全局规则

**配置方式**：Trae IDE → 设置 → 规则和技能 → 项目规则 → + 创建

**规则名称**：`coding-standards`（编码规范）

**显示名称**：`编码规范全局规则`

**生效方式**：始终生效

**规则内容**：
```markdown
# 编码规范全局规则

## 一、Python 编码规范

### 1.1 代码风格
- 遵循 PEP 8 编码规范
- 使用 4 个空格缩进，禁止使用 Tab
- 每行代码不超过 120 个字符
- 函数之间保留 2 个空行
- 类方法之间保留 1 个空行

### 1.2 命名规范
- 变量名：小写 + 下划线（如 stock_price）
- 函数名：小写 + 下划线（如 calculate_score()）
- 类名：首字母大写（如 ConfigManager）
- 常量：全大写 + 下划线（如 MAX_POSITION）
- 私有变量：单下划线前缀（如 _internal_var）

### 1.3 注释规范
- 所有函数必须包含中文 docstring
- 类必须包含类说明
- 复杂逻辑必须添加行内注释
- 注释格式：清晰、简洁、准确

## 二、智能体输出规范

### 2.1 代码输出
- 代码必须完整、可运行
- 禁止输出片段代码（除非特别说明）
- 代码块必须包含语言标识
- 代码后必须附带简要说明

### 2.2 文档输出
- 文档必须结构清晰、层次分明
- 重要内容使用粗体或表格强调
- 避免大段文字，使用列表分点
- 提供可操作的步骤和示例

### 2.3 错误处理
- 发现错误立即指出并说明原因
- 提供明确的修复方案
- 避免模糊表述（如"可能"、"也许"）
- 所有结论必须有依据

## 三、质量要求

### 3.1 代码质量
- 无语法错误
- 无逻辑错误
- 无安全漏洞
- 性能可接受
- 包含异常处理

### 3.2 文档质量
- 内容准确
- 表述清晰
- 结构完整
- 示例正确
- 格式统一
```

### 1.2 安全合规全局规则

**配置方式**：Trae IDE → 设置 → 规则和技能 → 项目规则 → + 创建

**规则名称**：`security-compliance`（安全合规）

**显示名称**：`安全合规全局规则`

**生效方式**：始终生效

**规则内容**：
```markdown
# 安全合规全局规则

## 一、代码安全

### 1.1 禁止行为
- 禁止硬编码敏感信息（密码、API Key、私钥等）
- 禁止使用不安全的函数（如 eval、exec）
- 禁止写入系统文件
- 禁止执行系统命令（除非明确需要且经过审批）
- 禁止网络请求（除非明确需要且经过审批）

### 1.2 数据处理
- 所有外部输入必须验证
- 文件操作必须包含异常处理
- 数据库操作必须防止注入
- 敏感数据必须加密存储

### 1.3 异常处理
- 所有外部调用必须包含 try-except
- 异常必须记录详细日志
- 禁止静默失败
- 关键错误必须抛出

## 二、交易安全

### 2.1 资金管理
- 必须设置最大仓位限制
- 必须设置止损策略
- 必须设置回撤控制
- 禁止全仓单只股票

### 2.2 风险控制
- 必须验证订单合法性
- 必须检查账户状态
- 必须设置交易频率限制
- 禁止异常交易行为

### 2.3 日志记录
- 所有交易必须记录日志
- 日志必须包含时间戳
- 日志必须包含操作详情
- 禁止删除或修改日志

## 三、数据安全

### 3.1 数据获取
- 必须使用官方 API 获取数据
- 必须验证数据完整性
- 必须处理数据异常情况
- 禁止使用未授权数据源

### 3.2 数据存储
- 本地数据必须加密存储
- 必须定期备份重要数据
- 禁止存储敏感用户信息
- 数据文件必须设置访问权限

### 3.3 数据使用
- 必须遵守数据使用协议
- 必须保护数据隐私
- 禁止数据泄露
- 禁止数据滥用
```

- ❌ 禁止使用中文字段名

### 3. 常用代码模式

#### 获取股票历史数据
```python
def get_stock_history(stock_code, start_date, end_date):
    """获取股票历史行情"""
    try:
        hist_data = get_history(
            security=stock_code,
            frequency='1d',
            start_time=start_date,
            end_time=end_date,
            fields=['open', 'high', 'low', 'close', 'volume']
        )
        
        if hist_data is None or hist_data.empty:
            log.warning(f"{stock_code} 历史数据为空")
            return None
        
        return hist_data
    except Exception as e:
        log.error(f"获取{stock_code}历史数据失败：{e}")
        return None
```

#### 获取财务数据
```python
def get_financial_data(stock_code):
    """获取股票财务数据"""
    try:
        growth = get_fundamentals(
            security=stock_code,
            table='growth_capacity',
            fields=['revenue_yoy', 'net_profit_yoy']
        )
        
        valuation = get_fundamentals(
            security=stock_code,
            table='valuation',
            fields=['pe_ttm', 'pb']
        )
        
        return {
            'growth': growth,
            'valuation': valuation
        }
    except Exception as e:
        log.error(f"获取{stock_code}财务数据失败：{e}")
        return None
```

- ✅ 返回 None 时必须处理
