# optimization_05 — 选股质量升级 + 持仓评分驱动调仓

> 创建日期：2026-05-09
> 基于：optimization_02 最优配置、optimization_03/04 全部失败实验、PTrade 2024/2025 最新运行交易明细
> 目标：在不破坏 2025 牛市收益能力的前提下，改善 2024 熊市/震荡市收益，减少无效换股，提升单笔交易质量
> 约束：PTrade API 兼容、分钟级回测、初始资金 10 万、策略文件仍为单文件 `backtest.py`

---

## 一、先给结论

optimization_05 不应继续沿着 optimization_03 和 optimization_04 的思路做参数扫描。

前四轮优化已经证明：

1. **继续优化市场状态识别，大概率只会继续伤害收益**
   - optimization_03 已系统证明：切换频率是 2025 收益的核心破坏因素。
   - 更复杂的熊市反弹/极端下跌子模式几乎零触发，复杂度显著增加但没有正收益。

2. **继续微调硬止损、轮换阈值、S 级权重、首日建仓等固定参数，也很难突破当前上限**
   - optimization_04 已系统证明：这些执行参数在当前框架下基本已经处于局部最优。
   - 单独把某个阈值从 `-12%` 改到 `-10%/-8%/-15%`、把轮换从 `-5%` 改到 `-8%/-10%`，都无法同时改善 2024 和 2025。

3. **真正还有空间的，不是“再调阈值”，而是“改变决策结构”**
   - 当前策略的核心问题已经从“仓位不对”转变为：
     - 买入候选质量不够稳定；
     - 调仓依据过于单一，只看“最弱持仓亏损多少”；
     - 卖出机制主要依赖固定阈值，缺少波动率、持仓阶段、候选替代价值三个维度；
     - 回测报表对“为什么卖、为什么换、换后是否更优”缺少可量化归因。

**因此，optimization_05 的核心方向应该是：**

`从“市场择时驱动”转向“持仓质量驱动”，从“固定阈值调参”转向“评分、分层、归因、约束”的结构升级。`

## 二、当前策略的真实状态

## 2.1 已被证明有效、应该保留的部分

以下能力不应在 optimization_05 中推倒重来：

1. **大盘模式检测只保留轻量版本**
   - `detect_market_mode_v2()` 的方向是对的：只用少量大盘指标，不做复杂打分。
   - 它更适合作为“仓位上限控制器”，而不是“高频切换驱动器”。

2. **sideways 80% / uptrend 90% / downtrend 30% 的仓位框架先保留**
   - optimization_02 和 optimization_04 已说明：继续在这组比例上做单参数扫描，收益空间极小，副作用大。

3. **S 级集中配置思路先保留**
   - `S=3.0, A/B=1.0` 在 optimization_04 中已被证实优于降权版本。
   - 后续如果要改，不应该直接调权重，而应该先提升 S 级识别准确率。

4. **早盘分支保留为可选增强，不作为主优化方向**
   - 09:35 早盘逻辑不是当前主要矛盾。
   - optimization_05 应优先解决“买什么、何时换、何时卖”的主问题。

## 2.2 已被证明无效、应停止投入的部分

1. 停止继续设计新的市场子模式。
2. 停止继续做 downtrend 进入/退出确认天数扫描。
3. 停止继续在固定硬止损阈值上做粗暴网格搜索。
4. 停止继续只看年度收益率而忽略换手、换股质量、卖出原因统计。

## 三、从当前代码看，真正需要优化的结构点

以下判断直接来自当前 `backtest.py` 的实现，而不是抽象猜测。

## 3.1 选股入口仍然偏“放量突破快照”，缺少质量分层

`SelectionAgent.select()` 当前仍以“四条件 + 打分”作为主入口：

- 满足 4/4 条件可进 S/A/B；
- 满足 3/4 条件可进 B；
- 满足 2/4 条件也可进 B；
- `MIN_SCORE_B` 虽然已经提高，但本质上仍属于“放宽准入、扩大候选池”的逻辑。

这套机制的问题不是“完全无效”，而是：

1. 对**趋势延续质量**刻画不足
   - 只看最近一个截面的突破、量能、MACD，不看过去 5~20 日的结构稳定性。

2. 对**负面因子**刻画不足
   - 没有显式惩罚高波动、过度乖离、长上影、反复假突破、量价背离。

3. 对**候选替代价值**没有输出
   - 当前只有入选分数，没有“持有中股票 vs 新候选股票”的统一可比较评分体系。

这意味着：

- 买入时可以排序；
- 但持有后无法回答“现在这只股票是否仍值得持有”；
- 轮换时只能退化成“谁亏得最多就先卖”。

## 3.2 当前轮换机制过于单因子

`ExecutionAgent.buy_new()` 中，轮换逻辑的核心是：

1. 找到最弱持仓；
2. 如果最弱持仓亏损超过阈值，则用新标的替换；
3. 替换动作只看亏损比例和新标的等级，不看：
   - 持仓天数；
   - 原持仓趋势是否破坏；
   - 新标的分数是否显著高于旧标的；
   - 当天是否已经发生过卖出；
   - 最近几天是否频繁轮换；
   - 当前组合是否已经高换手。

这会带来两个结果：

1. **亏损股被过早卖出**
   - 有些股票只是正常回撤，不应因为短期浮亏就被替换。

2. **调仓质量不可控**
   - 当前调仓是“损失控制型”，不是“机会成本最优化型”。

## 3.3 当前轮换资金分配存在实现风险

当前轮换使用的买入金额来自 `actual_cash_to_use`，该值是在卖出前按现有组合仓位计算的：

- `allowable_cash = target_total_value - current_pos_value`
- 当组合已接近目标仓位时，`actual_cash_to_use` 可能接近 0
- 但轮换本质上是“卖旧仓、买新仓”，理论上应使用卖出释放出来的资金，而不是卖出前剩余现金

这意味着当前轮换即使触发，也可能存在以下风险：

1. 买入金额明显偏小；
2. 卖出旧仓后未能充分回补新仓；
3. 组合暴露被无意中进一步降低；
4. 回测和实盘结果会被这个实现细节扭曲。

optimization_05 应把这一点作为**明确修复项**，不是参数项。

## 3.4 当前止损仍以固定阈值为主，缺少“股票自身波动适配”

虽然 optimization_04 已证明固定 `-12%` 好于 `-8%/-10%/-15%`，但这不等于 `-12%` 是最优逻辑，只能说明：

`在“固定阈值”这类方案里，-12% 是局部最优。`

真实问题是：

1. 对高波动领涨股，固定 `-12%` 可能过紧；
2. 对低波动阴跌股，固定 `-12%` 可能过松；
3. 时间止损 `30天/-5%` 仍然没有结合持仓质量变化；
4. 当前卖出缺少“阶段性管理”：
   - 建仓早期应该更重保护；
   - 趋势中段应该让利润奔跑；
   - 末端走弱时才应强化退出。

## 3.5 当前报表仍不足以支撑下一轮高质量优化

当前 `ReportAgent` 能输出账户报表和盈亏统计，但 optimization_05 需要新增更有用的归因数据：

1. 每笔卖出的**原因分类**
   - 硬止损 / 时间止损 / MA 跌破 / 峰值回撤 / 轮换卖出 / 移动止盈

2. 每次轮换的**前后对比**
   - 被卖持仓的持仓评分、持仓天数、亏损比例
   - 新候选的入场评分、排名、入场类型
   - 轮换后 5 日 / 10 日 / 20 日相对收益

3. 候选池质量统计
   - 当日候选数
   - S/A/B 占比
   - 候选平均分 / 前 5 名分差
   - 实际买入标的在候选池中的排名

没有这些数据，后续仍会陷入“看年度收益变化猜原因”的低效循环。

## 四、optimization_05 的总体方案

## 4.1 核心原则

optimization_05 采用以下四条原则：

1. **不推翻当前轻量市场框架**
   - 保留 `detect_market_mode_v2()` 作为仓位上限控制器。

2. **把优化重点从“市场状态识别”转到“持仓质量评估”**
   - 决策核心从 market regime 切换为 holding quality。

3. **把调仓从“亏损触发”升级为“评分差触发”**
   - 允许适度轮换，但必须有明确的机会优势。

4. **把止损从“固定阈值”升级为“固定阈值 + 波动率 + 持仓阶段”**
   - 先保护资金，再保护利润，最后保护趋势。

## 4.2 新版本的策略结构

optimization_05 建议把策略升级为四层结构：

### 层 1：市场暴露层

用途：

- 只决定组合总仓位上限；
- 不直接决定单只股票去留；
- 不再参与高频调仓。

建议：

- 保留当前 `sideways / trending_uptrend / trending_downtrend` 三档；
- 暂不扩展新子模式；
- 仅修复明显逻辑问题，例如 `recent_5d_return < -0.02` 与 `< -0.03` 的重复分支。

### 层 2：入场评分层

用途：

- 决定哪些股票可以新开仓；
- 输出统一的 `entry_score` 与 `entry_rank`；
- 严格控制低质量 B 级股票进入组合。

### 层 3：持仓评分层

用途：

- 对已持有股票按日更新 `holding_score`；
- 评估股票是否仍属于“组合前列持仓”；
- 为轮换与卖出提供统一依据。

### 层 4：退出与轮换层

用途：

- 卖出理由标准化；
- 调仓条件结构化；
- 限制过度换手。

## 五、具体开发方案

## 5.1 模块 A：选股评分升级为 2.0

### 目标

提高新开仓质量，减少“能买但不值得买”的 B 级标的进入组合。

### 设计思路

在现有四维打分基础上，引入“正向因子 + 负向惩罚 + 持续性确认”。

### 建议新增特征

1. **趋势持续性**
   - `RS20`: 个股 20 日涨幅相对沪深 300 的超额收益
   - `RS60`: 个股 60 日涨幅相对沪深 300 的超额收益
   - `MA20 slope`: 近 5 日 MA20 斜率

2. **量价结构质量**
   - 近 5 日平均成交额 / 近 20 日平均成交额
   - 最近 10 日阳线占比
   - 最近 10 日突破后回踩次数

3. **拥挤度/延展度惩罚**
   - 收盘价相对 MA20 乖离过大时扣分
   - 单日长上影占比过大时扣分
   - 近 5 日振幅过大时扣分

4. **候选稳定性**
   - 连续 2 天进入候选池加分
   - 前一日是候选但今日显著降级则扣分

### 准入规则建议

1. `S/A` 仍可来自 4/4 条件；
2. `B` 仅允许来自：
   - 3/4 条件且综合分足够高；
   - 或 2/4 条件但必须满足更强的相对强度与趋势持续性；
3. 在 `trending_downtrend` 中默认关闭 “2/4 -> B”；
4. 引入 `MAX_NEW_B_POSITIONS`，限制 B 级新开仓数量。

### 代码改造建议

在 `SelectionAgent` 中新增：

- `compute_entry_features(stock, df, df_index)`
- `score_entry_v2(features, mode)`
- `classify_entry_grade_v2(score, features, mode)`

保留旧逻辑开关：

```python
FEATURE_FLAGS = {
    "ENTRY_SCORE_V2": False,
}
```

这样可保证 optimization_05 分阶段回测，不与旧逻辑混在一起。

## 5.2 模块 B：持仓评分引擎

### 目标

把“当前持仓中谁最应该继续持有”量化出来，替代纯亏损视角。

### 持仓评分建议

对每只持仓每天计算 `holding_score`，范围可设为 `0~100`。

建议构成：

1. **趋势健康度 40%**
   - close > MA20 / MA30 / MA60
   - MA20 斜率是否向上
   - 最近 10 日新高次数

2. **相对强度 20%**
   - 持仓股近 10/20/40 日相对沪深 300 超额收益

3. **盈利质量 20%**
   - 浮盈是否为正
   - 峰值回撤是否可控
   - 盈利是否在扩大而非收缩

4. **筹码稳定性 20%**
   - 近 5 日振幅
   - 量价是否恶化
   - 是否跌破关键均线

### 用法

1. 不是所有低分持仓都立即卖；
2. 低分持仓只代表“可以进入轮换候选池”；
3. 真正卖出还要结合新候选分数差、持仓天数、当日卖出预算。

### 状态存储建议

新增全局状态：

```python
g._holding_scores = {}
g._sell_reasons = []
g._rotation_history = []
```

## 5.3 模块 C：轮换机制升级为“评分差驱动”

### 目标

做到“适当调仓换股”，而不是“机械止损式换股”。

### 新规则

只有同时满足以下条件，才允许轮换：

1. 当前组合已满仓位槽位；
2. 当天尚未超过 `MAX_ROTATIONS_PER_DAY`；
3. 被替换持仓持有天数 >= `MIN_HOLD_DAYS_FOR_ROTATION`；
4. 被替换持仓 `holding_score` 低于阈值；
5. 新候选 `entry_score - holding_score` 大于 `ROTATION_SCORE_DELTA`；
6. 被替换持仓满足至少一个弱化条件：
   - 跌破 MA20/MA30；
   - 相对强度转负；
   - 峰值回撤超过阈值；
   - 连续 N 天评分下降；
7. 当前市场不是极端防御日。

### 关键改动

把轮换规则从：

`最弱持仓亏损 > 阈值 -> 替换`

升级为：

`旧持仓质量显著弱化 + 新候选显著更优 -> 替换`

### 重要限制

1. 每日最多轮换 1 只；
2. 同一只股票卖出后 `N` 天内禁止回补；
3. 新买入股票最少持有 `M` 天后才能进入轮换池；
4. 若当日已触发硬止损或时间止损，则禁止额外主动轮换。

### 资金修复

轮换买入金额必须改为基于**卖出释放资金**计算，而不是使用卖出前现金。

建议做法：

1. 先记录待卖持仓当前市值；
2. 预估可用买入金额 = 待卖市值 * `(1 - fee_buffer)`；
3. 执行卖单；
4. 再用预估金额买入新标的；
5. 若框架支持成交后回调补单，则可在回调中二次校准。

这是 optimization_05 的必做修复项。

## 5.4 模块 D：卖出机制升级为“三阶段退出”

### 目标

保留当前有效的风控思想，但把退出逻辑从单一固定阈值升级为更适配趋势股的机制。

### 阶段划分

#### 阶段 1：建仓保护期（0~5 个交易日）

目标：防止刚买入就变成错误交易。

建议规则：

1. 使用更紧的初始保护：
   - 固定止损与 ATR 止损二选一取更严格值；
   - 例如 `max(-0.08, -2.2 * ATR10 / entry_price)`。
2. 不做主动轮换，除非出现明确失效。

#### 阶段 2：趋势持有期（6~25 个交易日）

目标：让强股尽量跑。

建议规则：

1. 减弱固定硬止损影响，转为：
   - MA20/MA30 跟踪；
   - 峰值回撤；
   - 持仓评分持续下降；
2. 对强趋势股尽量不做 L1/L2 止盈，避免卖飞；
3. 对弱趋势但浮盈明显的股票，可以保留部分止盈。

#### 阶段 3：衰退退出期（26 个交易日以上）

目标：清理占用仓位但已失去进攻性的股票。

建议规则：

1. 若浮盈不高且持仓评分持续低位，则触发时间退出；
2. 若已经大幅盈利但趋势破坏，则走移动止盈；
3. 若跌破中期均线且相对强度明显转弱，允许优先退出并释放仓位。

### 为什么这比继续调 `-12%` 更合理

因为 optimization_04 已经证明：

- 固定阈值之间的差异只是在不同年份间搬运损失；
- 但三阶段退出会根据持仓所处阶段改变保护方式，更符合趋势股生命周期。

### PTrade 兼容性说明

不依赖外部技术指标库。

ATR 可以直接基于日线 OHLC 用 pandas/numpy 手工计算：

```python
tr = max(high-low, abs(high-prev_close), abs(low-prev_close))
atr = tr.rolling(10).mean()
```

## 5.5 模块 E：报表与归因体系升级

### 目标

让每一轮优化都能回答“为什么变好/变坏”，而不是只看年度收益率。

### 必增输出

1. **sell_reason_report.csv**
   - 日期
   - 股票
   - 卖出原因
   - 持仓天数
   - 卖出时收益率
   - 卖出时 holding_score

2. **rotation_report.csv**
   - 日期
   - 被卖股票 / 新买股票
   - 被卖收益率
   - old holding_score
   - new entry_score
   - score_delta
   - 5/10/20 日相对收益

3. **candidate_quality_report.csv**
   - 日期
   - 候选总数
   - S/A/B 数量
   - Top1~Top5 分数
   - 实际成交标的排名

### 新验收指标

optimization_05 不再只看 `2024收益 / 2025收益`，还要同时看：

1. 年度换手次数
2. 主动轮换次数
3. 被动止损次数
4. 平均持仓天数
5. 单笔平均盈亏
6. 盈亏比
7. 卖出后 10 日反弹比例
8. 调仓后 10 日超额收益

## 六、建议的开发顺序

optimization_05 必须分阶段实施，禁止一次性大改。

## Phase 1：只升级选股评分

### 目标

在不改卖出逻辑和买入结构的前提下，提高入场质量。

### 任务

1. 新增 `ENTRY_SCORE_V2`；
2. 先只影响新买入候选排序；
3. `ENTRY_SCORE_V2` 第一轮只纳入 3 类排序增强因子：
   - `RS60`：用中期相对强弱拉开高质量候选之间的排序；
   - `NEGATIVE_PENALTY`：对冲高回落、偏离短均线过大、量价背离等弱信号做减分；
   - `CANDIDATE_PERSISTENCE`：对连续多日进入候选池但尚未成交的高质量票做小幅加分；
4. 不改轮换，不改止损，不改仓位，不改每日买入数量。

### 边界

本阶段严格限定为“排序增强”，不做以下动作：

1. 不调整 `S/A/B` 分级准入门槛；
2. 不限制 `B` 级开仓数量；
3. 不引入候选质量门控；
4. 不改变 `ExecutionAgent.buy_new()` 的资金分配逻辑；
5. 不提前触碰 `Phase 2/Phase 3` 的轮换与退出逻辑。

### 验收

1. 2025 收益不能明显低于基线；
2. 2024 年低质量 B 级开仓数下降；
3. 候选 Top5 与实际成交的质量分差变大；
4. 若单阶段回测未通过，不进入 `Phase 2`。

### 实施细化

#### 1. 配置层

在 `ConfigManager` 中新增但默认关闭：

```python
FEATURE_FLAGS = {
    "ENTRY_SCORE_V2": False,
}

ENTRY_SCORE_V2 = {
    "ENABLE_NEGATIVE_PENALTY": True,
    "ENABLE_RS60": True,
    "ENABLE_CANDIDATE_PERSISTENCE": False,
    "RS60_WEIGHT": 0.05,
    "NEGATIVE_PENALTY_WEIGHT": 0.18,
    "MAX_NEW_B_POSITIONS": None,  # Phase 1 不启用
}
```

说明：

1. 本阶段只允许打开 `ENTRY_SCORE_V2`；
2. `ROTATION_V2`、`EXIT_STAGE_V1` 必须保持关闭；
3. `MAX_NEW_B_POSITIONS` 明确保留但不启用，后续如需启用必须先更新本文档；
4. `CANDIDATE_PERSISTENCE` 在当前 `Phase 1` 子方案中明确关闭，不作为排序因子。

#### 2. 选股层

本阶段只改 `SelectionAgent`，推荐新增/改造以下函数：

1. `compute_entry_features()`
2. `score_entry_v2()`
3. 在 `select()` 中保留原有 `S/A/B` 准入，再对已入围候选计算 `entry_score_v2`

推荐实现边界：

1. `compute_entry_features()` 只提取排序特征，不参与准入判断；
2. `score_entry_v2()` 只输出排序加减分，不改变原始 `base_score` 的准入含义；
3. `select()` 中候选排序改为优先按 `entry_score_v2` 排序，其次按 `base_score` 排序；
4. `selected` 继续保持 `(stock, score, grade)` 兼容结构，避免影响 `ExecutionAgent.buy_new()`。

#### 3. 三类排序增强因子

当前 `Phase 1` 子方案只允许实现以下 2 类因子：

1. `RS60`
   - 计算个股近 60 日涨幅相对基准指数的超额收益；
   - 用于在高质量候选之间拉开排序，不用于淘汰候选；
   - 当前子方案中必须显著降低权重，只允许作为弱加分项，不能主导排序。
2. `NEGATIVE_PENALTY`
   - 对冲高回落、偏离 `MA5/MA20` 过大、跌回 `MA20` 下方等弱信号做减分；
   - 只减排序分，不改变 `S/A/B` 等级。

禁止新增未在本文档确认的第四类因子。

#### 4. 报表与观测

在不改变现有增强报表主结构的前提下，`candidate_quality_report` 建议补充：

1. `top1_base_score`
2. `top2_base_score`
3. `top3_base_score`
4. `entry_score_v2` 排序后的 `selected_symbols`

目的：

1. 对比原始分数与 V2 排序分是否出现明显漂移；
2. 判断 Top5 候选与最终成交之间的质量分差是否扩大。

#### 5. 实施步骤

严格按以下顺序执行：

1. 增加 `ENTRY_SCORE_V2` 配置，默认关闭；
2. 增加 `compute_entry_features()`；
3. 增加 `score_entry_v2()`；
4. 只在 `SelectionAgent.select()` 中接入排序增强；
5. 保持买入、轮换、止损、退出逻辑完全不动；
6. 补充 `candidate_quality_report` 里的基础分/排序分观测字段；
7. 先跑 `2025Q1` 冒烟；
8. 再跑 `2025` 全年；
9. 只有 `2025` 不明显退化，才跑 `2024` 全年；
10. 单阶段未通过则停止，不进入 `Phase 2`。

### Phase 1 子方案 A（2026-05-10）

基于本轮失败样本，当前确认采用 `A 保守修订` 继续验证 `Phase 1`。子方案边界如下：

1. `2024` 是否回到正收益区间附近；
2. `2024` 的硬止损密度是否下降；
3. `2025` 是否仍能保持对稳定基线的相对优势。

### Phase 3 归因分析（2026-05-11）

#### 核心结论

`Phase 3 子方案 A` 的 `EXIT_STAGE_V1` 三条退出路径（`atr_protect_stop`、`atr_trend_stop`、`decay_exit`）在 2024 和 2025 全年回测中**触发次数均为零**。这不是”轻微改善被统计波动抵消”，而是**退出增强完全没有产生可观测行为变化**。全年结果与稳定基线完全一致（2025: +14.43%, 2024: +11.76%）。

#### 证据来源及可靠性分级

> **可靠性分级说明**：以下证据分为三级——**最终日志确认**（每轮回测独立生成日志文件，可严格定量）；**辅助观察**（`sell_reason_report.csv` 等报表为跨轮次 append 累积文件，未按 run 隔离，可能混合多轮数据，仅作参考）；**代码审查确认**（直接审查源码，可靠）。

1. 2025 全年日志 `backtest_250101_251231_260511_093657.log`：`atr_protect_stop` / `atr_trend_stop` / `decay_exit` 关键词出现次数均为 0 — **最终日志确认**
2. 2024 全年日志 `backtest_240101_241231_260511_093734.log`：同上，均为 0 — **最终日志确认**
3. 增强报表 `sell_reason_report.csv`：卖出原因分布为 hard_stop(328)、time_stop(169)、rotation(119)、low_holding_score_exit(22)，无任何 ATR 或 decay 退出 — **辅助观察**（累积报表，非单轮隔离数据）
4. 代码审查 `backtest.py:2239-2400`：EXIT_STAGE_V1 的接入点位于 hard_stop/time_stop 之后、峰值回撤/MA 跟踪之前 — **代码审查确认**

#### 逐机制根因

##### 1. atr_protect_stop：ATR 保护线比硬止损更宽，被硬止损优先截断

**触发条件**（`backtest.py:2339-2356`）：
- `stage == “protect”`（持仓 ≤ 5 天）
- `close < buy_price - ATR * 2.2` 且 `profit_ratio < 0`

**代码执行顺序**：
1. 硬止损检查（`-12%`）→ 若触发则 `continue`，跳过 EXIT_STAGE_V1
2. EXIT_STAGE_V1 → `atr_protect_stop` 检查

**根因**：`trends_up` 策略选出的趋势股本身波动较大（有量能突破、MACD 金叉等特征），典型 ATR 约为价格的 3-6%。

- `ATR * 2.2 = 6.6% ~ 13.2%` 的保护距离
- 当 ATR > 5.45% 时，`ATR * 2.2 > 12%`，保护线低于 `-12%` 硬止损线
- 硬止损在代码中优先级更高，永远先于 ATR 保护触发

**数据证据**（以下统计来自 `sell_reason_report.csv`，该报表为跨轮次累积文件，数字仅作辅助参考）：
- 102 次 protect 阶段 hard_stop（持仓 ≤ 5 天），亏损范围 `-12.3%` 到 `-20.1%`，中位数 `-15.3%`
- 这些 position 的 holding_score 中位数 60.49（仍然健康），说明下跌发生得非常突然
- 无任何一次在 `-4%` 到 `-12%` 区间被 ATR 保护截断

##### 2. atr_trend_stop：trailing stop 跌破时利润已不满足 > 3% 条件

**触发条件**（`backtest.py:2358-2373`）：
- `stage == “trend”`（持仓 6-25 天）
- `profit_ratio > 0.03`
- `close < peak_price - ATR * 2.2`

**根因**：逻辑设计层面的矛盾。

- `trailing_stop = peak_price - ATR * 2.2`
- 对于 ATR = 5% 的股票，从入场涨 10% 后：
  - `trailing_stop = entry * 1.10 - entry * 0.05 * 2.2 = entry * 0.99`（低于入场价！）
  - 当价格跌到 trailing_stop 时，`profit_ratio ≈ -1%`，不满足 `> 3%` 条件
- 只有当 `ATR < (peak - entry * 1.03) / 2.2` 时，trailing stop 才可能在 `profit > 3%` 时触发
- 对于 10% 涨幅：`ATR < 3.18%`。趋势股 ATR 通常 > 3%，条件几乎不可能同时满足

**结论**：trailing stop 的宽度（`ATR * 2.2`）使得止损位远低于入场价，而前置条件要求利润 > 3%，两者在趋势股上形成不可调和的矛盾。

##### 3. decay_exit：三重 AND 条件在极窄时间窗口内不可能同时满足

**触发条件**（`backtest.py:2375-2400`）：
- `stage == “decay”`（持仓 > 25 天）
- `holding_score < 40`
- `profit_ratio < -0.03`
- `close < MA20`

**根因**：

1. **时间窗口极窄**：decay 阶段从 day 26 开始，但 time_stop 在 day 31 触发（`> 30` 天，`< -5%`）。只有 4 个交易日的窗口
2. **holding_score 不够低**：protect 阶段 hard_stop 的 holding_score 中位数 60.49，说明趋势股在快速下跌时评分仍然健康。到 day 26 时，部分 position 的 holding_score 可能仍在 40 以上
3. **与 time_stop 重叠**：从数据看，time_stop 的 holding_score 中位数 24.61（远 < 40），说明这些股票在 day 31+ 时评分已很低。但 day 26-30 时可能评分还未充分恶化
4. **与 hard_stop 重叠**：decay 阶段的 hard_stop（day 26-30）亏损中位数约 -15%，远超 -3% 的 decay 阈值。这些 position 在触发 decay_exit 之前已被 hard_stop 截断

**数据证据**（以下统计来自 `sell_reason_report.csv` 累积报表，仅作辅助参考）：
- decay 窗口（days 26-30）的卖出事件仅有 hard_stop（亏损 > -12%）和低评分退出，无 decay_exit
- time_stop（days 31+）的 holding_score 中位数 24.61，确认这些 position 在更晚的时间点才充分恶化

#### 综合诊断

`EXIT_STAGE_V1` 的三条退出路径全部因为**设计层面的结构性问题**而完全未生效：

1. **protect 路径**：ATR 保护线比硬止损更宽或相当，被硬止损优先截断 → 需要让 ATR 保护在硬止损之前检查，或者缩小 ATR 乘数
2. **trend 路径**：trailing stop 宽度与 `profit > 3%` 前置条件存在逻辑矛盾 → 需要重新设计 trailing stop 的计算方式或移除 profit 前置条件
3. **decay 路径**：三重 AND 条件在 4 天窗口内不可能同时满足 → 需要放宽条件或扩展时间窗口

#### 对后续执行者的建议

1. **优先碰什么**：
   - 重新设计 protect 阶段的止损位置：考虑将 ATR 保护放在硬止损之前检查（调整代码优先级），或者使用更小的 ATR 乘数（如 1.5 而非 2.2）
   - 重新设计 trend 阶段的 trailing stop：移除 `profit > 3%` 前置条件，或改用 `peak_drawdown` 方式（从峰值回撤 N% 即触发，不与入场价比较）
   - 放宽 decay 条件：改为二选一（`holding_score < 40 OR profit < -3%`），而非三重 AND

2. **不该碰什么**：
   - 不应该只做参数微调（如把 ATR_MULTIPLIER 从 2.2 改到 2.0），因为核心矛盾是结构性的
   - 不应该移除现有硬止损，它仍然是最后的防线
   - 不应该在未解决重叠问题的情况下增加更多退出条件

3. **设计原则**：
   - 任何新的退出条件必须确保与现有卖出机制（hard_stop、time_stop、peak_drawdown、MA_stop）之间**不存在逻辑死区或重叠覆盖**
   - 必须在代码中明确退出检查的优先级顺序，并在文档中说明为什么这个顺序是合理的
   - 验收标准必须包含”退出触发次数 > 0”这一最低门槛

## Phase 1 子方案 A 宏观归因分析（2026-05-11）

### 为什么它是当前唯一通过双年份验证的方案

`Phase 1 子方案 A` 的核心改动极其克制：**仅改变已入围候选的排序优先级，不改变准入逻辑、不改变买卖决策结构、不改变仓位框架。** 它只做了两件事：

1. **`RS60`（权重 0.05）**：在高质量候选之间，用中期相对强弱做微弱排序增强
2. **`NEGATIVE_PENALTY`（权重 0.18）**：对"冲高回落、偏离均线过远、跌回 MA20 下方"的弱信号做减分

这组设计恰好命中了当前策略最薄弱的环节：**候选入围之后的"挑谁"质量**。

### 改善主要来自哪里

以下判断基于 Phase 1 子方案 A 与稳定基线的行为对比（均为单模块验证，不改准入、不改卖出）：

1. **更少低质量 B 级票被排到前排**：
   - `NEGATIVE_PENALTY`（权重 0.18）是决定性因子。它对"距 MA5/MA20 过远"、"冲高回落后未完成整理"、"已跌回 MA20 下方"的惩罚，恰好拦截了那些"满足四条件但结构已走坏"的伪突破股。
   - 这类股在稳定基线中会以 B 级身份排到买入队列前排并被成交，随后快速触发硬止损或时间止损。`Phase 1 子方案 A` 把它们压到候选池后排甚至掉出 Top N，资金流向更健康的候选。

2. **2024 年硬止损密度下降**（间接推断）：
   - 2024 熊/震荡市中，伪突破比例远高于 2025 牛市。`NEGATIVE_PENALTY` 在 2024 年的拦截效果更明显，因为弱信号候选更多、惩罚幅度更大。
   - 这解释了为什么 2024 从 `+11.76%` → `+15.23%`（+3.47pp），改善来自"少买了会亏的票"。

3. **2025 年候选前排质量拉开**：
   - 牛市中伪突破少，但候选数量多。`RS60`（权重 0.05）在 5 只 S 级候选竞争 3 个买入槽位时，帮策略挑到了相对更强的标的。
   - 这解释了为什么 2025 从 `+14.43%` → `+32.12%`（+17.69pp），改善来自"在强市中挑到了更强的"。

4. **CANDIDATE_PERSISTENCE 关闭是关键**：
   - 子方案 A 明确关闭了 `CANDIDATE_PERSISTENCE`。这避免了在弱环境中对"连续出现但未成交"的个股形成追高偏误——初版 `Phase 1` 的 2024 崩塌（`-20.54%`）很可能与此有关。

### 当前剩余损失主要来自哪里

即使 `Phase 1 子方案 A` 已通过，以下损失源仍未解决：

1. **B 级票仍然可以进入组合**：
   - 当前准入逻辑：`3/4` → B，`2/4` → B。这些路径在 `Phase 1` 中完全未被触碰。
   - `ENTRY_SCORE_V2` 只改变已入围 B 级票之间的排序，不阻止低质量 B 级票入围。
   - 在弱市中，`2/4 → B` 路径仍是"满足两个条件就能买"的宽松准入。

2. **弱市下候选池不收缩**：
   - 当前无论市场是 `trending_uptrend`、`sideways` 还是 `trending_downtrend`，候选池准入标准完全一致。
   - 在 `trending_downtrend` 中，即使"满足 4/4 条件"的股票，其后续表现也远差于牛市中的同类。
   - 但准入逻辑不做任何市场环境适配。

3. **候选与成交之间的质量差距约束缺失**：
   - 当前只要候选排名进入 Top N 就买入，不考虑 Top1 与 Top N 之间的分数差距。
   - 如果候选池整体质量不高（弱市常见），Top1 到 Top5 分数都很低且差距很小，策略仍会满仓买入。没有"候选池质量不够时不买"的约束。

4. **B 级持仓数量无上限**：
   - 配置中 `MAX_NEW_B_POSITIONS` 已预留但设为 `None`（不限制）。
   - 在弱市中，B 级票积累可能导致组合整体质量下沉。

### 下一轮最值得动的单一杠杆

**结论：准入分层 + 弱市收缩（买得更少、挑得更严、只在值得买时买）。**

理由：

1. **这是 `Phase 1 子方案 A` 已验证方向的自然延伸**。`Phase 1` 证明了"在不改准入的前提下靠排序提升质量"有效。下一步应该升级到"在弱市改准入标准"。

2. **这是当前唯一还有显著改善空间的决策环节**。Phase 2（轮换）和 Phase 3（退出）已明确失败。市场模式方向（optimization_03/04）也已证明复杂化有害。只有买入端还没有被充分挖掘。

3. **风险可控**。准入收紧只会减少弱市入场的次数和质量，不会改变已持仓股票的去留逻辑（硬止损、时间止损、MA 跟踪止损等基线保护仍在）。

4. **不会伤害 2025**。牛市环境下候选质量高，准入收紧的边际影响很小——4/4 S/A 级候选足够多，不会因为收紧 B 级而买不满。

## Phase 1B：准入分层 + 弱市收缩

> **定位**：Phase 1B 是 `Phase 1 子方案 A` 的直接延续，是 optimization_05 的 **下一步唯一允许推进的主线**。
>
> **前提**：Phase 2（轮换）已失败，Phase 3（退出）已封档，Phase 4 已暂停。当前唯一有效方向是买入端结构升级。

### 目标

在不破坏 `Phase 1 子方案 A` 已验证收益结构的前提下，进一步降低低质量入场、提高候选池质量、在弱市中主动收缩而非被动承受。具体目标：

1. **2024 年继续改善**（在 `+15.23%` 基础上进一步提升，底线是不退化）
2. **2025 年不明显低于 Phase 1 子方案 A**（允许轻微波动，底线是不低于基线 `+14.43%`）
3. **B 级入场数量和质量显著改善**：
   - 弱市（`trending_downtrend` / `sideways`）中 `2/4 → B` 路径关闭或严格受限
   - B 级持仓占组合比例下降
4. **候选池质量可观测提升**：候选 Top1~Top3 分差扩大，成交标的在候选池中的排名更靠前

### 开关边界

### 只允许改哪些内容

1. **B 级准入分层**：
   - `3/4 → B`：保留但可提高最低分门槛
   - `2/4 → B`：按市场模式区分——仅在 `trending_uptrend` 中保留，在 `sideways` 和 `trending_downtrend` 中关闭

2. **弱市候选收缩**：
   - 在 `trending_downtrend` 中，候选池准入条件整体收紧（如要求至少满足 3/4 条件，关闭 2/4 路径）

   - 激活已有配置项 `MAX_NEW_B_POSITIONS`，限制组合中 B 级票的最大数量

   - `candidate_quality_report` 中新增：`b_admitted_by_2of4`（通过 2/4 路径入围的 B 级数量）、`b_admitted_by_3of4`（通过 3/4 路径）、`market_mode`（当前市场模式）、`quality_gap_triggered`（是否因质量差距约束跳过买入）

### 建议配置项

在现有 `ENTRY_SCORE_V2` 配置块中新增以下字段：

```python
ENTRY_SCORE_V2 = {
    # Phase 1 子方案 A 保留项
    "ENABLE_NEGATIVE_PENALTY": True,
    "ENABLE_RS60": True,
    "ENABLE_CANDIDATE_PERSISTENCE": False,
    "RS60_WEIGHT": 0.05,
    "NEGATIVE_PENALTY_WEIGHT": 0.18,
    "PERSISTENCE_BONUS_PER_DAY": 0.015,
    "PERSISTENCE_MAX_BONUS": 0.06,

    # Phase 1B 新增项
    "MAX_NEW_B_POSITIONS": 2,                # B 级持仓上限（从 None 激活为 2）
    "MIN_SCORE_B_SIDEWAYS": 0.52,            # sideways 中 B 级最低准入分（在 base_score 尺度上，≥ 当前 0.45 的 ~15% 收紧）
    "MIN_SCORE_B_DOWNTREND": 0.60,           # downtrend 中 B 级最低准入分（在 base_score 尺度上，≥ 当前 0.55 的 ~9% 收紧）
    "ALLOW_2OF4_B_IN_SIDEWAYS": False,       # sideways 中关闭 2/4 → B 路径
    "ALLOW_2OF4_B_IN_DOWNTREND": False,      # downtrend 中关闭 2/4 → B 路径
    "ALLOW_2OF4_B_IN_UPTREND": True,         # uptrend 中保留 2/4 → B 路径
    "MIN_CANDIDATE_SCORE_GAP": 0.08,         # Top1 与买入候选分差阈值（entry_score 尺度）
    "CANDIDATE_GAP_SKIP_REDUCE": 1,          # 质量差距不足时减少买入数量（非全部跳过）
}
```

说明：
- `MIN_SCORE_B_SIDEWAYS=0.52` / `MIN_SCORE_B_DOWNTREND=0.60`：弱市中 B 级票即使满足 3/4 条件，其 base_score 也必须达到该阈值才能入围。此阈值与 `score_breakout_stock()` 的 base_score 使用同一尺度（约 0.40-0.85），比对应的 `MIN_SCORE_B`（0.45/0.55）分别收紧约 15% 和 9%。牛市不设额外最低分。
- `ALLOW_2OF4_B_IN_SIDEWAYS=False` / `ALLOW_2OF4_B_IN_DOWNTREND=False`：弱市中 `2/4 → B` 路径完全关闭，只保留 `3/4 → B`。
- `MIN_CANDIDATE_SCORE_GAP=0.08`：若 Top1 候选与第 N 名候选的分数差距 < 0.08，减少 1 只买入（`CANDIDATE_GAP_SKIP_REDUCE=1`），而非全部跳过。这样在候选质量不够时降低暴露，但不完全放弃交易机会。
- `MAX_NEW_B_POSITIONS=2`：组合中最多同时持有 2 只 B 级票。超出时，即使有空余槽位也只买 S/A 级，不买 B 级。

## 收口补强：run_id 唯一性修复 + Phase 1A 同口径对照（2026-05-11）

### run_id 唯一性修复

**问题**：原 `run_id` 由 `start_date + 到秒时间戳` 组成，同起始日、同秒启动的两个回测可能碰撞。

**修复**：在 `initialize()` 中为 run_id 追加 `os.getpid()` 和 `os.urandom(2).hex()`（4 位随机十六进制），使 collision 概率降至极低。

**修改后格式**：`{start_date}_{timestamp}_{pid}_{rand4hex}`，例如 `240101_260511_213411_22053_458e`。

**影响**：仅影响增强报表文件名，不影响策略行为和回测结果。LOW 风险。
