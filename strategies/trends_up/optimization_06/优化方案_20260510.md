## 一、为什么需要 `optimization_06`

`optimization_04` 与 `optimization_05` 已经把当前策略的主要结构矛盾暴露得很清楚：

1. `optimization_04` 的优势是 **2025 牛市弹性极强**：
   - `2024`：`-0.97%`
   - `2025`：`+62.24%`
   - 结论：策略在牛市中具备很强的赚钱能力，但在熊市/弱市中防守不足。

2. `optimization_05` 的优势是 **双年份更加稳健**：
   - 默认配置 `Phase 1B`
   - `2024`：`+15.23%`
   - `2025`：`+32.12%`
   - 结论：策略在弱市中的稳定性大幅提升，但牛市弹性明显低于 `optimization_04`。

3. `optimization_05` 还已经证明了什么方向**不该再继续投入**：
   - `Phase 2 / ROTATION_V2`：失败
   - `Phase 3 / EXIT_STAGE_V1`：失败
   - `Phase 4`：无可组合通过模块，已暂停

4. 因此，`optimization_06` 的核心问题不再是：
   - “怎么卖得更聪明”
   - “怎么轮换得更积极”
   - “怎么继续抠买入排序细节”

5. `optimization_06` 的真正问题是：
   - 如何在 **不丢掉 `optimization_05` 防守能力** 的前提下，
   - 重新找回接近 `optimization_04` 的 **2025 牛市收益弹性**。

一句话总结：

**`optimization_06` 的任务，是把 `opt04` 的牛市进攻能力和 `opt05` 的熊市防守能力，尽可能在同一单策略框架中拼起来。**

## 二、当前已知事实

### 2.1 已验证有效的事实

1. **买入端优化有效**
   - `Phase 1 子方案 A` 与 `Phase 1B` 均已通过。
   - 说明“买谁”这个问题已经明显优于老基线。

2. **`Phase 1B` 的作用是风险收缩，不是收益增强**
   - 它让行为更收缩、候选更保守、默认配置更稳。
   - 但它没有把 `2025` 从 `+32.12%` 再往上推。

3. **退出和轮换不是当前主线**
   - 继续投入 `EXIT_STAGE_V1` 或 `ROTATION_V2` 已被事实证明无效。

4. **增强报表隔离问题已修复**
   - 后续所有正式回测必须继续使用 run_id 隔离报表。

### 2.2 已验证失败的事实

1. 仅靠交易执行参数优化，无法同时改善 `2024` 和 `2025`
2. 仅靠退出增强，无法在双年份稳定成立
3. 仅靠继续收紧准入，不能把 `2025` 拉回 `+62.24%`

### 2.3 由此得到的结论

下一步不能再围绕：

- 卖出逻辑
- 主动轮换
- 单纯准入继续收紧
- 市场模式检测重写

而必须转向：

**组合层的非对称暴露控制。**

## 三、`optimization_06` 的核心假设

### 3.1 核心假设

当前收益张力并不主要来自“选错票”，而是来自：

1. **强机会期没有把优势放大到足够程度**
   - 高质量 `S/A` 机会虽然能选出来，但组合暴露未必足够进攻。

2. **弱机会期仍然保留了过多低效暴露**
   - 即使 `Phase 1B` 已经收缩了 B 级准入，组合层仍可能“有槽位就买”，没有充分体现“机会质量低时主动留现金”的思想。

3. **同一套固定仓位/买入节奏，无法同时匹配 2024 和 2025**
   - `2024` 需要更强的防守与现金保留；
   - `2025` 需要更强的高质量机会放大能力。

### 3.2 新主线

`optimization_06` 采用如下主线：

**非对称组合暴露 / 仓位分层 / 机会密度驱动的买入强度控制**

翻译成策略语言就是：

- 强机会环境：敢买、敢集中、敢把高质量候选放大
- 弱机会环境：少买、慢买、宁可留现金也不勉强补仓位

## 四、总目标与设计原则

### 4.1 最终目标

`optimization_06` 的最终验收目标明确写死如下：

1. `2024` 全年分钟级收益率 **>= `+15.23%`**
2. `2025` 全年分钟级收益率 **>= `+62.24%`**

这两个目标**必须同时满足**，才算最终成功。

### 4.2 设计方式

本轮采用：

**先拆分，后合并**

即：

1. 先做 `P0` 归因实验，确认 2025 收益差的主因
2. 再设计 `2025` 进攻子线
3. 再设计 `2024` 防守子线
4. 最后只组合双方都证明副作用可控的杠杆

### 4.3 重要约束

即使本轮出现阶段性 gap，也**不接受“当前单策略框架下不可兼得”作为终点**。

这意味着：

- 文档必须内置 gap 追踪机制
- 未达标时必须继续迭代
- 不允许一句“做不到”就结束

### 4.4 防返工原则

为了避免前几轮那种来回试错，本轮必须遵守：

1. 每一轮只验证一个清晰杠杆
2. 每条子线先单独验证，再允许合并
3. 任一子线若在自己的主目标年份都拉不动，立即淘汰
4. 任一子线若把另一年打穿底线，立即淘汰
5. 合并阶段只允许组合“单独有效且副作用有限”的杠杆
6. 不允许边回测边追加新规则
7. 如需新增测试，必须先更新本文档，再执行

## 六、总体架构：P0 归因 + 双子线 + 合并验证

`optimization_06` 拆成四段推进：

1. **P0：候选池构成与收益归因实验**
2. **子线 A：2025 进攻杠杆**
3. **子线 B：2024 防守杠杆**
4. **子线 C：合并验证**

### 6.1 P0：候选池构成与收益归因实验

目标：

- 在正式启动 `A/B` 子线前，先确认 `2025` 从 `+62.24%` 降到 `+32.12%` 的主因，到底来自：
  - `ENTRY_SCORE_V2` 的排序变化
  - `Phase 1B` 的准入收缩
  - 还是两者叠加

为什么必须先做：

1. 如果 2025 收益差主要来自**候选池构成变化**，那么子线 A 只做“更高暴露”大概率方向错误。
2. 如果高弹性标的已经在 `Phase 1B` 下被显著过滤，那么“买更多”只是在更保守的池子里加大下注，无法有效追回 2025 收益。
3. 只有先搞清楚收益差的来源，后续才能判断应优先做：
   - `A1` 暴露增强
   - `A2` 强趋势期候选弹性恢复
   - 或两者依次验证

P0 最少必须输出：

1. `opt04` 与 `Phase 1A/1B` 在同一段牛市窗口下的 `S/A/B` 候选分布差异
2. 实际买入标的的风格差异
3. 高弹性标的是否被 `ENTRY_SCORE_V2` 或 `Phase 1B` 过滤掉
4. `2025` 收益差的主因排序

P0 的直接后果：

1. 若主因是**暴露不足**，子线 A 以 `A1 暴露增强` 为主。
2. 若主因是**候选池过于保守**，子线 A 必须补 `A2 强趋势期候选弹性恢复`。
3. 若两者都有贡献，则子线 A 采用“双段式”推进，但仍坚持单杠杆逐个验证。

### 6.2 子线 A：2025 进攻杠杆

目标：

- 在不明显破坏 `2024` 的前提下，尽可能恢复 `2025` 的牛市弹性

关注点：

- 强机会环境是否买得不够积极
- 高质量 `S/A` 候选是否没有被足够集中放大
- 单日买入数量和资金分配是否过于保守
- 候选池本身是否已经被“去弹性化”

### 6.3 子线 B：2024 防守杠杆

目标：

- 在不明显压制 `2025` 的前提下，尽可能维持甚至增强 `2024` 的防守收益

关注点：

- 弱机会日是否仍买得过满
- 候选质量不足时是否仍然补齐仓位
- B 级占比是否仍导致组合整体质量下沉

### 6.4 子线 C：合并验证

目标：

- 把 `A` 和 `B` 中单独有效且副作用可控的杠杆组合起来
- 只做“可解释、可回退、可验证”的组合

重要纪律：

- 不允许把极端进攻杠杆与极端防守杠杆硬拼
- 不允许为了追目标，把多个未经验证的小改动打包一起上

## 七、主杠杆定义

### 7.1 本轮唯一主杠杆

`optimization_06` 的唯一主杠杆是：

**非对称组合暴露 / 仓位分层**

它具体包含三类可合法操作的杠杆：

1. **总仓位上限随环境变化**
2. **单日新开仓数量随机会密度变化**
3. **S/A/B 级资金分配强度随环境变化**
4. **强趋势期的候选弹性恢复**

### 7.2 什么叫“机会密度”

本轮中的“机会密度”，统一指：

1. 当日候选池中高质量 `S/A` 级候选的数量
2. 前排候选之间的分数差距
3. 候选池整体分布是否呈现“前强后弱”还是“整体都平”
4. 当前市场模式下这些候选是否值得放大暴露
5. 当前机会属于“牛市高弹性窗口”还是“熊市反弹假强势”

### 7.3 核心思想

不是“有槽位就买满”，而是：

- 候选强、分差大、`S/A` 充足：多买，集中买
- 候选弱、分差小、`B` 占比高：少买，保留现金
- 只有市场模式与候选质量同时确认，才允许触发进攻性暴露

## 八、允许改哪些内容

本轮只允许修改以下范围：

### 8.1 组合层暴露控制

允许：

1. 不同市场模式下的 **目标总仓位上限**
2. 不同市场模式下的 **每日最大新开仓数量**
3. 不同市场模式下的 **现金保留比例**

### 8.2 等级驱动的买入力度

允许：

1. `S/A/B` 等级对应的资金权重
2. 高质量候选占优时的集中度放大
3. `B` 级候选占比偏高时的主动减仓位强度

### 8.3 机会密度驱动的买入节奏

允许：

1. 根据 `candidate_quality_report` 的前排质量差距决定当日买几个
2. 根据 `S/A` 候选数量决定当日是否提高或降低暴露
3. 在弱机会日“主动不买满”

### 8.4 增强报表观测项补充

允许增加与本轮验证直接相关的最小观测字段，例如：

1. 当日目标仓位
2. 当日实际买入数量
3. 当日 `S/A/B` 新开仓数量
4. 当日因机会密度不足而主动留现金的标志
5. 当日因环境收缩导致未买满的原因

### 8.5 强趋势期候选弹性恢复

允许：

1. 仅在高置信强趋势环境下，对部分进攻性准入约束做适度放宽
2. 比较“仅暴露增强”与“暴露增强 + 候选弹性恢复”的差异
3. 将候选弹性恢复视为子线 A 的次级杠杆，而不是默认启用项

## 九、明确禁止改哪些内容

以下内容在 `optimization_06` 中**全部禁止**：

1. **禁止重开退出机制方向**
   - 不改 `EXIT_STAGE_V1`
   - 不改任何新的止盈止损模块

2. **禁止重开轮换机制方向**
   - 不改 `ROTATION_V2`
   - 不新增新的换仓判定逻辑

3. **禁止重写市场模式检测框架**
   - `detect_market_mode_v2()` 只能读取，不能重写

4. **禁止改硬止损 / 时间止损 / 既有卖出结构**
   - 所有 `PositionAgent` 卖出主逻辑不动

5. **禁止引入多策略组合**
   - 本轮仍是单策略框架，不拆成“震荡策略 + 趋势策略”

6. **禁止引入 ML 或黑盒模型**
   - 不新增机器学习选股、回归模型、外部因子引擎

7. **禁止大杂烩式同时改多个维度**
   - 每次只验证一个清晰杠杆

8. **禁止把 `opt04` 的旧执行逻辑直接整体搬回**
   - 本轮不是回退，而是结构性合成新版本

9. **禁止只依赖 `detect_market_mode_v2()` 单一信号做激进/防守切换**
   - 所有暴露增强必须同时经过候选质量和机会密度确认

## 十、子线 A：2025 进攻杠杆设计

### 10.1 子线 A 目标

在不把 `2024` 打穿到 `+15.23%` 以下的前提下，尽量恢复 `2025` 的进攻弹性。

### 10.2 子线 A 的两段结构

子线 A 分成两段：

1. **A1：暴露增强**
   - 解决“明明是强机会环境，但买得不够积极”的问题
2. **A2：强趋势期候选弹性恢复**
   - 解决“候选池本身已经过于保守，导致高弹性票被过滤”的问题

执行纪律：

1. 先做 `A1`
2. 只有当 `A1` 无法显著缩小 `2025 gap`，且 `P0` 归因表明候选池构成是主因时，才允许进入 `A2`

### 10.3 子线 A1：暴露增强允许尝试的杠杆

只允许从以下方向中逐个验证：

1. **强趋势期提高总仓位上限**
   - 当模式为 `trending_uptrend` 且 `S/A` 候选充足时，允许更高总仓位

2. **强趋势期提高单日新开仓数量**
   - 当机会密度足够高时，不再受当前偏保守的“少买”节奏限制

3. **强化 `S` 级权重**
   - 但只能在强趋势/高质量候选条件下启用

4. **高质量机会日提高集中度**
   - 当 Top1~Top3 明显强于后排时，允许集中度更高

### 10.4 子线 A2：强趋势期候选弹性恢复

只允许在 `P0` 明确证明“候选池过于保守是 2025 收益差主因”后，才可尝试：

1. 在高置信强趋势窗口中，适度恢复部分高弹性候选的进入概率
2. 对 `ENTRY_SCORE_V2` 或 `Phase 1B` 中最可能压制高弹性票的约束做条件性放宽
3. 该放宽必须是：
   - 只在强趋势窗口生效
   - 可完全回退
   - 可在增强报表中被观测

禁止做法：

1. 不能全局放松准入
2. 不能把 `Phase 1B` 全局回退到 `opt04`
3. 不能在未完成 `P0` 的情况下直接做弹性恢复

### 10.5 子线 A 不能做什么

1. 不能通过放松 `B` 级准入来追 `2025`
2. 不能通过关闭 `Phase 1B` 防守项整体回退到 `opt04`
3. 不能通过更激进卖出/止盈来伪造收益改善

### 10.6 子线 A 的主评估指标

1. `2025` 全年收益提升幅度
2. `2025` 最大回撤是否仍可接受
3. `2025` 的 `S/A` 主导程度是否增强
4. `2024` 是否被明显打穿
5. 若进入 `A2`，必须额外说明高弹性候选恢复是否真实发生

## 十一、子线 B：2024 防守杠杆设计

### 11.1 子线 B 目标

在不明显压制 `2025` 的前提下，确保 `2024 >= +15.23%`，并尽可能降低弱机会期的无效暴露。

### 11.2 子线 B 允许尝试的杠杆

只允许从以下方向中逐个验证：

1. **弱趋势/震荡期降低总仓位上限**
2. **弱趋势/震荡期降低单日新开仓数量**
3. **当日候选质量不足时主动留现金**
4. **当 `B` 占比过高时进一步降低实际开仓数量**

### 11.3 子线 B 不能做什么

1. 不能通过更严格卖出止损来伪造防守
2. 不能把所有弱市都压成接近空仓，导致 `2025` 中后段机会错失
3. 不能直接回到 `opt05` 更保守以外的无差别全局收缩

### 11.4 子线 B 的主评估指标

1. `2024` 全年收益和最大回撤
2. `2024` 的 hard_stop 密度是否下降
3. 弱机会日是否更少出现“候选质量差但仍买满”
4. `2025` 是否被明显压制

## 十二、合并验证设计

### 12.1 合并前提

只有当某个杠杆满足以下条件时，才允许进入合并阶段：

1. 在自己的目标年份中确实带来正向改善
2. 对另一年份的副作用有限且可解释
3. 行为变化可通过增强报表被清楚观察
4. 逻辑简单、可回退、边界明确

### 12.2 合并原则

1. 一个进攻杠杆 + 一个防守杠杆，优先
2. 不组合两个都很激进的杠杆
3. 不组合两个都强依赖市场模式判定边缘值的杠杆
4. 暴露切换必须采用“市场模式 + 候选质量 + 机会密度”的联合门控
5. 合并后仍必须能明确解释“为什么有效”

### 12.3 合并后的目标

合并阶段唯一目标：

- 同时满足 `2024 >= +15.23%` 与 `2025 >= +62.24%`

## 十五、gap 追踪机制

### 15.3 重要说明

虽然本轮不接受“不可兼得”作为终点，但也不允许无约束乱试。

1. 这轮主要缩小哪一侧 gap

## 十六、失败后的继续迭代规则

因为本轮必须持续迭代直到达标，所以失败后的处理必须标准化。

### 16.1 单杠杆失败

1. 标记为“高副作用杠杆”
2. 禁止直接进入合并阶段

若合并后仍未同时达标：

1. 回退到最近通过的单杠杆状态
2. 更新本文档记录本轮 gap 变化
3. 只允许继续尝试下一组候选杠杆
4. 不允许重新打开已封档的 `Phase 2/3/4`
