## 二、optimization_01 失败经验总结

### 2.1 已尝试的方案及失败原因

| Phase | 方案 | 2024收益 | 2025收益 | 失败原因 |
|-------|------|----------|----------|---------|
| Phase 1 | MAX_POSITIONS扫描(5~8) | -10%~-5% | — | 无行情识别，bull期仓位带入bear期 |
| Phase 2 T1 | 静态标注+防御性仓位 | +10.6% | **+78.6%** | 2024不够(需>30%)，静态标注有前视偏差 |
| Phase 2 T2 | 静态标注+均衡仓位 | +47.9% | +18.3% | 2025不够(需>50%)，sideways仓位敏感 |
| Phase 3 | 动态行情识别(6指标评分) | +23.3% | **+0.4%** | 切换14次(Phase 2仅3次)，频繁切换=纯损失 |
| Phase 3+v3 | 动态+参数优化(17组合) | +27.4% | -1.2% | 参数调整对2025无效，瓶颈在切换频率 |
| Phase 3.5 | 降低切换频率(5天确认) | -32.2% | +8.0% | 降低灵敏度对2024伤害更大 |

### 2.2 五大核心教训

#### 教训1：切换频率是2025收益的决定性因素

| 方案 | 2025切换次数 | 2025收益 |
|------|-------------|----------|
| Phase 2 静态标注 | 3次 | **+78.6%** |
| Phase 3.5 (5天确认) | 7次 | +8.0% |
| Phase 3 (3天确认) | 14次 | +0.4% |

**结论**：每多切换一次，2025收益下降约5-10%。任何优化都不能增加切换频率。

#### 教训2：2024-2025存在不可调和的tradeoff

```
2024有真实bear期 → 需要防御(低仓位/快切换) → 2024收益好但2025差
2025无bear期(静态标注) → 不需要防御 → 切换=纯损失

Phase 2 T1: 防御强 → 2024=10.6%(不够), 2025=78.6%
Phase 2 T2: 防御弱 → 2024=47.9%(好), 2025=18.3%(不够)
```

**结论**：不能通过简单调参解决，需要结构性改变。

#### 教训3：动态检测的"正确性"悖论

动态检测在2025年1月和4月识别出的bear信号是**真实的**（CSI300确实下跌），但这些是**短期回调**而非趋势性下跌。策略的防御响应在参数层面是正确的，但在**时机层面是错误的**——过度防御导致错过反弹。

**结论**：不要试图精确识别bear期，而是用更简单的机制（硬止损）来保护。

#### 教训4：参数优化在错误层面工作

- `calc_target_ratio()` 覆盖了 `REGIME_PARAMS.TARGET_POS_RATIO`，v1优化脚本的defense参数**完全无效**
- defense_floor参数9组覆盖，2025收益恒定0.3%，说明bear期仓位不是2025的瓶颈
- sideways_base参数3组覆盖，2025收益恒定0.4%，说明sideways仓位也不是瓶颈

**结论**：不要在仓位映射函数上做参数优化，瓶颈在其他地方。

#### 教训5：Phase 2的78.6%依赖前视偏差

Phase 2 T1的2025年78.6%收益依赖静态标注"2025年无bear期"这一结论。这是基于全年数据的事后判断，**实盘中无法获得**。

**结论**：不能依赖静态标注，但可以借鉴其"少切换"的核心思想。

### 2.3 从失败中提炼的成功要素

尽管所有方案都失败了，但有几个**正向信号**值得保留：

1. **Phase 2 T2的2024年+47.9%**：说明在bear期适度防御+bull期积极进攻是可行的
2. **Phase 2 T1的2025年+78.6%**：说明"少切换+让利润奔跑"在牛市中极其有效
3. **Phase 3的2024年+23.3%**：动态检测在熊市中有一定防御能力
4. **MAX_POSITIONS=5的2025年+60.93%**（持仓数实验）：小资金集中持仓在牛市中表现优异

## 三、2024年亏损根因分析

### 3.1 核心数据

- 全年仅 **3 个交易日** 有交易活动（1月2~4日）
- 5只持仓持有 **360+天** 未卖出
- **417次** "资金不足/放弃买入"记录
- **零止损触发**，**零止盈触发**
- 市场模式：trending_downtrend 占 **98.3%**（238天）

### 3.2 因果链

```
1月2日建仓5只（满仓）→ 1月初股市下跌 → 持仓浮亏 → 
检测到trending_downtrend → 但MA120止损太慢（股价已远低于MA120）→ 
止损从未触发 → 资金锁死 → 每天选股但无法买入 → 策略瘫痪
```

### 3.3 根本缺陷

1. **无硬止损**：仅有MA120均线止损，在快速下跌中形同虚设
2. **市场模式检测滞后**：用持仓盈亏判断市场，已亏损后才知道是熊市
3. **建仓过激**：首日满仓98%，不留缓冲
4. **无持仓时间止损**：亏损股可以无限期持有

## 四、优化方案（4大类）

### 4.1 硬止损机制 ⭐⭐⭐ 优先级最高

**目标**：解决2024年"冻死"问题，防止单只股票深套

| 参数 | 当前值 | 优化值 | 说明 |
|------|--------|--------|------|
| 绝对止损线 | 无 | **-12%** | 从买入价亏损超12%强制卖出，不分市场模式 |
| 持仓时间止损 | 无 | **30天/-5%** | 持仓超30天且浮亏>5%强制卖出 |
| downtrend MA止损 | MA120 | **MA60** | 下跌趋势中用更快的均线止损 |

### 4.2 市场模式检测优化 ⭐⭐⭐

**目标**：提前感知市场环境变化，避免"已亏损才知道是熊市"

| 检测维度 | 当前 | 优化后 |
|---------|------|--------|
| 主信号 | 持仓盈亏（滞后） | 沪深300 MA20 vs MA60（领先） |
| 辅助信号 | 无 | 沪深300 近5日涨跌幅 |
| 切换确认 | 无 | **连续3天**新信号才切换（避免频繁切换） |
| downtrend仓位上限 | 60% | **30%** |

### 4.3 选股漏斗优化 ⭐⭐

**目标**：提高选股质量，减少"选到垃圾股"的概率

| 参数 | 当前值 | 优化值 | 说明 |
|------|--------|--------|------|
| MIN_TOTAL_CAPITAL | 30亿 | **50亿** | 排除小盘垃圾股 |
| MAX_TOTAL_CAPITAL | 200亿 | **200亿** | 保持不变（用户约束） |
| MIN_SCORE_B | 0.20 | **0.45** | 提高B级准入门槛 |
| 成交量持续性 | 无 | **近5日>20日×1.2** | 新增过滤条件 |
| 相对强度 | 无 | **近20日>沪深300** | 新增过滤条件 |

### 4.4 仓位管理优化 ⭐⭐

**目标**：避免首日满仓锁死，保留调仓灵活性

| 参数 | 当前值 | 优化值 | 说明 |
|------|--------|--------|------|
| 首日建仓上限 | 5只（满仓） | **3只** | 留40%现金作为缓冲 |
| uptrend仓位上限 | 98% | **90%** | 适度保守 |
| downtrend仓位上限 | 60% | **30%** | 更激进的防御 |
| 现金保留线 | 无 | **10%** | 始终保留10%现金 |

## 五、迭代结果

### 5.1 完整迭代历史

| 迭代 | 2024收益率 | 2025收益率 | 2024最大回撤 | 2025最大回撤 | 说明 |
|------|-----------|-----------|-------------|-------------|------|
| 基准 | -18.67% | +62.24% | -37.03% | -14.53% | 无优化 |
| 迭代1 | -18.67% | +62.24% | -37.03% | -14.53% | Phase 1+2完整优化（无效） |
| 迭代2 | -16.29% | +31.14% | -25.48% | -11.95% | sideways仓位80%→50% |
| 迭代3 | -22.74% | +62.24% | -40.84% | -14.53% | 快速downtrend确认（1天） |
| 迭代4 | **-0.97%** | **+62.24%** | -38.63% | -14.53% | 降低downtrend阈值（最佳2024） |
| 迭代5 | -2.53% | +36.99% | -24.71% | -13.34% | sideways仓位60%+硬止损-8% |
| 迭代6 | -12.37% | +47.98% | -28.05% | -18.49% | 首日建仓1只+硬止损-10% |

### 5.2 关键发现

#### 2024年最佳：-0.97%（迭代4）
- sideways仓位保持80%（2025的关键）
- downtrend触发阈值降低（-3%→-2%，7%回撤）
- downtrend仅需1天确认

#### 2025年最佳：+62.24%（迭代1和迭代4）
- 保持sideways仓位80%
- 避免过度限制仓位

#### trade-off分析
- **降低sideways仓位会伤害2025**：迭代2和迭代5都显示2025收益大幅下降
- **首日建仓限制会伤害2024**：迭代6显示首日仅1只导致2024亏损加剧
- **硬止损过严（-8%）会伤害2025**：迭代5的2025从62%降至37%

### 5.3 2024年目标未达成原因

**目标：>30%，最佳结果：-0.97%，差距31%**

深层原因：
1. **2024年是典型熊市**：沪深300下跌约35%，策略在熊市中难以盈利
2. **市场模式检测滞后**：即使优化后downtrend触发30次，仍无法避免亏损
3. **选股质量问题**：2024年选到太多高风险股票，即使硬止损也亏损
4. **sideways市场139天（57%）**：这部分市场无法通过仓位管理改善

### 5.4 下一步优化方向（待探索）

| 方向 | 方案 | 问题 | 风险 |
|------|------|------|------|
| 方向A | 更激进硬止损（-12%→-8%） | -12%仍让亏损累积 | 可能过度交易 |
| 方向B | sideways仓位80%→60% | 2024年sideways市场139天(57%) | 可能伤害2025收益 |
| 方向C | 首日建仓3只→1只 | 2024年1月建仓过于激进 | 可能伤害2024 |
| 方向D | MIN_SCORE_B 0.45→0.55 | 选到太多高风险股票 | 减少交易机会 |

## 十一、结论
