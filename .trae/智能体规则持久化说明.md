# 智能体规则持久化说明

**创建日期**: 2026-03-27  
**规则版本**: v2.0  
**持久化位置**: `.trae/agents_rules.md`

---

## 背景

在之前的对话中，用户多次提到需要更新智能体规则，但发现智能体在后续任务中会"忘记"这些规则。为了解决这个问题，需要将所有规则持久化到一个固定的配置文件中，并要求所有智能体在每次任务开始前必须阅读和遵守。

---

## 持久化方案

### 1. 规则文件位置

**文件路径**: `SimTradeLab/.trae/agents_rules.md`

**选择理由**:
- `.trae/` 是 Trae IDE 的配置目录，智能体会自动识别
- `agents_rules.md` 命名清晰，易于查找
- 位于项目根目录，所有智能体都能访问

### 2. 规则内容

规则文件包含以下核心章节:

#### 2.1 核心原则
- 必须使用多智能体协作模式
- 保持策略名称稳定（不加版本号后缀）
- 文档组织有序
- 代码备份机制
- **持久化规则**（新增）：所有智能体必须在每次任务开始前读取本规则文件

#### 2.2 文档保存规则（重点强调）
- QA 文档保存位置（优先级排序）
- 优化文档保存位置
- 策略说明文档位置
- 正确示例和错误示例对比

#### 2.3 智能体职责与规则
- **Strategy-Arch**: 职责、输出目录、禁止目录、必须遵守的规则
- **Strategy-Engr**: 职责、输出目录、禁止目录、代码备份规则、必须遵守的规则
- **Strategy-QA**: 职责、输出目录、禁止目录、QA 文档命名规范、必须遵守的规则

#### 2.4 快速检查清单（每次任务必读）
- 智能体自检问题（5 个关键问题）
- 文档保存位置速查表
- 常见错误及纠正措施

#### 2.5 违规处理机制
- 自动检查和纠正步骤
- 用户反馈处理流程
- 规则更新历史

### 3. 执行机制

#### 3.1 智能体责任

**所有智能体必须**:
1. 在每次任务开始前读取 `.trae/agents_rules.md`
2. 在任务执行过程中严格遵守规则
3. 在任务完成后检查是否符合规则
4. 发现错误立即纠正

#### 3.2 违规后果

如果智能体违反规则:
- **文档位置错误**: 删除错误文件，重新保存到正确位置
- **命名错误**: 立即更正
- **未备份代码**: 回退并重新执行备份
- **用户指出错误**: 承认错误 → 说明违反的规则 → 立即纠正 → 更新规则文件

#### 3.3 用户监督

用户可以通过以下方式监督:
1. 检查文档是否保存在正确位置
2. 检查策略命名是否规范
3. 检查代码是否已备份
4. 发现错误时明确指出违反的规则

## 规则要点总结

### 核心规则（必须记住）

1. **多智能体协作**: Strategy-Arch + Strategy-Engr + Strategy-QA
2. **策略命名**: 保持 `trends_up` 不变，不加 `_v2`, `_v3` 等后缀
3. **文档位置**:
   - Arch: `docs/`, `tasks/`, `strategies/{name}/optimize_XX/optimization_plan.md`
   - Engr: `strategies/{name}/optimize_XX/backtest.py`
   - QA: `strategies/{name}/optimize_XX/qa_report.md` (优先) 或 `workspace_qa/{name}/`
4. **代码备份**: 每次优化前必须备份到 `backtest_backup_YYYYMMDD.py`
5. **目录整洁**: QA 文档禁止放在 `workspace_qa/` 根目录

### 文档保存优先级

| 文档类型 | 最优位置 | 次优位置 | 禁止位置 |
|---------|---------|---------|---------|
| 优化方案 | `strategies/{name}/optimize_XX/` | - | `workspace_qa/` |
| QA 报告 | `strategies/{name}/optimize_XX/qa_report.md` | `workspace_qa/{name}/` | `workspace_qa/` 根目录 |
| 策略说明 | `strategies/{name}/策略说明_Readme.md` | - | `docs/` |
| 回测分析 | `strategies/{name}/optimize_XX/` | `workspace_qa/{name}/` | `workspace_qa/` 根目录 |

## 如何确保智能体不忘记

### 1. 规则文件设计特点

- ✅ **位置固定**: `.trae/agents_rules.md`，智能体容易找到
- ✅ **内容完整**: 包含所有重要规则和执行机制
- ✅ **结构清晰**: 分章节、有示例、有对比
- ✅ **易于查阅**: 快速检查清单、速查表、常见错误

### 2. 智能体行为约束

- **任务前**: 必须读取规则文件，回答自检问题
- **任务中**: 严格遵守规则，不确定时查阅规则
- **任务后**: 检查是否符合规则，发现错误立即纠正

### 3. 持续改进机制

- 每次违反规则后，更新规则文件，避免再次犯错
- 规则版本号递增（v1.0 → v2.0 → ...）
- 记录规则更新历史

## 验证方法

### 验证智能体是否记住规则

**测试问题**:
1. QA 文档应该保存在哪里？
   - 正确答案：`strategies/{name}/optimize_XX/qa_report.md` (优先) 或 `workspace_qa/{name}/`

2. 策略名称可以带版本号吗？
   - 正确答案：不可以，保持 `trends_up` 不变

3. 优化代码前需要做什么？
   - 正确答案：先备份到 `backtest_backup_YYYYMMDD.py`

4. QA 文档可以放在 `workspace_qa/` 根目录吗？
   - 正确答案：不可以，禁止

### 检查清单

在智能体完成任务后，检查:
- [ ] 文档是否保存在正确位置
- [ ] 策略命名是否规范
- [ ] 代码是否已备份
- [ ] QA 文档是否避免了根目录
- [ ] 文件名是否简洁（不带版本号）
