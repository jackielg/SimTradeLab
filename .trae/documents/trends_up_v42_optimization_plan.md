# Trends_Up 策略 v4.2 优化计划

## 📋 任务概述

**目标**：分析 trends_up v4.1 fix 策略在 2025-04-01 ~ 2025-07-31 周期亏损 -32.76% 的根本原因，并实施优化方案。

**约束条件**：
- 必须使用 Strategy-Arch, Strategy-Engr, Strategy-QA 多智能体协作模式
- 保持策略名称为 trends_up（不加版本号后缀）
- 所有文档保存在 `strategies/trends_up/optimize_02/` 目录
- 优化前备份当前 backtest.py

---

## 🎯 优化目标

### 收益目标
- 总收益率 > 50%（3 个测试周期中至少 2 个达标）
- 年化收益率 > 60%

### 风控目标
- 最大回撤 < 15%（3 个测试周期全部达标）
- 盈亏比 > 1.5
- 胜率 > 55%

### 测试周期
1. **震荡市**：2025-04-01 ~ 2025-07-31（已测试：-32.76% ❌）
2. **牛市**：2024-09-01 ~ 2024-12-31（待测试）
3. **完整年**：2024-01-01 ~ 2024-12-31（待测试）

---

## 🔍 第一阶段：深度诊断分析（Strategy-QA 主导）

### 1.1 回测数据详细分析

**任务描述**：
- 分析 83 天的每日交易数据
- 识别亏损的主要来源
- 统计胜率、盈亏比、持仓周期等关键指标

**具体步骤**：
1. 读取回测日志文件（backtest_250401_250731_260328_005250.log）
2. 提取所有交易记录（买入/卖出）
3. 计算每笔交易的盈亏
4. 分析亏损交易的共同特征

**输出**：
- 交易统计表（盈利/亏损交易数量、平均盈亏幅度）
- 亏损交易特征分析（持仓天数、止损原因、市场状态）
- 盈利交易特征分析

### 1.2 选股逻辑分析

**任务描述**：
- 检查选股评分系统是否有效
- 分析选出的股票后续表现
- 识别选股模型的缺陷

**具体步骤**：
1. 检查 TrendStrengthScorer 类的评分逻辑
2. 统计选股评分与后续表现的相关性
3. 分析高分股票是否真的表现更好
4. 检查评分阈值（0.35 分）是否合理

**输出**：
- 选股评分有效性分析
- 评分阈值优化建议
- 评分权重调整建议

### 1.3 退出策略分析

**任务描述**：
- 分析 ATR 止损、时间止损、止盈策略的执行情况
- 识别退出策略的问题

**具体步骤**：
1. 统计各种退出原因的数量和盈亏
2. 分析 ATR 止损参数（2.0 倍）是否合理
3. 检查止盈层级（15%/25%/45%/70%）是否科学
4. 评估时间止损（25 天/20 天/15 天）是否过早

**输出**：
- 退出策略效果分析
- ATR 参数优化建议
- 止盈/止损参数优化建议

### 1.4 市场环境判断分析

**任务描述**：
- 验证市场状态判断的准确性
- 分析仓位管理是否合理

**具体步骤**：
1. 检查 MarketEnvironmentAnalyzer 类的判断逻辑
2. 统计 83 天中各市场状态的天数
3. 验证市场状态判断是否与后续走势一致
4. 评估仓位管理参数（牛市 15%、震荡市 10%、熊市 5%）

**输出**：
- 市场状态判断准确性分析
- 仓位管理参数优化建议

### 1.5 生成诊断报告

**输出文档**：
- `strategies/trends_up/optimize_02/diagnosis_report_v41.md`

**内容包括**：
- 亏损根本原因（3-5 个核心问题）
- 问题优先级排序
- 优化建议清单

## 🏗️ 第二阶段：优化方案设计（Strategy-Arch 主导）

### 2.1 制定优化方案

**任务描述**：
- 基于诊断报告，设计 v4.2 优化方案
- 确定优化的优先级和范围

**优化方向**（根据诊断结果确定）：
1. **选股逻辑优化**
   - 调整评分权重
   - 增加新的评分维度（如资金流、板块热点）
   - 优化评分阈值

2. **退出策略优化**
   - 调整 ATR 止损参数
   - 优化止盈层级
   - 改进时间止损逻辑

3. **仓位管理优化**
   - 调整市场状态判断逻辑
   - 优化仓位计算公式
   - 增加动态调仓机制

4. **风险控制优化**
   - 增加单行业集中度限制
   - 增加相关性检查
   - 优化最大回撤控制

**输出文档**：
- `strategies/trends_up/optimize_02/optimization_design.md`

### 2.2 设计评审

**参与智能体**：
- Strategy-Arch：方案架构评审
- Strategy-QA：可行性评审
- Strategy-Engr：实现难度评估

**评审要点**：
- 优化方案是否针对核心问题
- 预期效果是否明确
- 实现复杂度是否可控
- 是否引入新的风险
